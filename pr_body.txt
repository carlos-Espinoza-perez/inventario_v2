## Issue relacionado
Closes #45, Closes #46, Closes #47, Closes #48, Closes #49

## Causa encontrada
- #45: La pantalla del monitor de caja no retornaba a la vista anterior tras abrir la caja exitosamente.
- #46: Los campos de login no tenian indicaciones nativas de autocompletado (AutofillHints).
- #47: Los campos del formulario de creacion de productos no avanzaban el foco secuencialmente.
- #48: La opcion de imprimir etiquetas aparecia marcada por defecto incluso para productos que ya tenian un codigo de barras real guardado en la base de datos.
- #49: El modal de tallas solo mostraba (S, M, L) al fallar la lectura del key correcto en el JSON de especificaciones.

## Solucion aplicada
- Se agrego `context.pop()` en la validacion de apertura exitosa del turno si es posible retornar.
- Se envolvio el formulario de login en un `AutofillGroup` y se anadieron `AutofillHints.email` y `.password`.
- Se configuro `TextInputAction.next` y `FocusScope.of(context).nextFocus()` en autocompletados y campos de formulario.
- Se lee la base de datos para verificar si la variante posee un SKU comercial no-generado (diferente a 'GEN-'); si es asi, se deshabilita la impresion por defecto. Tambien se deshabilita si se escanea mediante la camara dentro del modal.
- Se lee el JSON de categoria bajo la llave `tallas_permitidas` y se combinan con tallas ya existentes en DB.
