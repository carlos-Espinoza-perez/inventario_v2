🎯 OBJETIVO DE ESTE MÓDULO
===========================================================================
Cargar el modelo Gemma 2B (descargado en el paso 01) en la memoria del 
dispositivo utilizando MediaPipe LLM Inference API. El objetivo es lograr 
respuestas en tiempo real aprovechando la GPU del móvil de forma asíncrona.

===========================================================================
🏗️ ESTRATEGIA DE CARGA (ISOLATES)
===========================================================================
Un LLM de 1.5GB puede "congelar" el hilo principal de Flutter (UI) durante 
la inicialización. 

ESTRATEGIA:
1. Inicializar el motor en un 'Isolate' (hilo separado).
2. Configurar la aceleración por GPU (obligatorio para Gemma 2B).
3. Establecer el 'Context Window' a 2048 tokens para ahorrar RAM.



===========================================================================
💻 CÓDIGO DE IMPLEMENTACIÓN (DART)
===========================================================================

import 'package:mediapipe_genai/mediapipe_genai.dart';
import '01_setup_assets_manager.dart';

class AIService {
  LlmInference? _llmInference;
  final AIModelManager _modelManager = AIModelManager();

  // 1. Inicialización del Motor
  Future<void> initializeEngine() async {
    if (_llmInference != null) return;

    final modelFile = await _modelManager.getModelFile();
    
    if (!await modelFile.exists()) {
      throw Exception("El modelo no existe. Debe descargarse primero.");
    }

    // Configuración optimizada para gama media
    final options = LlmInferenceOptions(
      modelPath: modelFile.path,
      maxTokens: 1024,      # Limitar longitud de respuesta para velocidad.
      temperature: 0.2,     # Baja temperatura para respuestas técnicas y precisas.
      randomSeed: 42,
      topK: 40,
    );

    // Carga asíncrona en GPU
    _llmInference = await LlmInference.create(options);
  }

  // 2. Método de Inferencia (Prompting)
  Future<String> generateResponse(String prompt) async {
    if (_llmInference == null) await initializeEngine();

    try {
      // Generación de respuesta (Streaming opcional)
      final response = await _llmInference!.generateResponse(prompt);
      return response;
    } catch (e) {
      return "Error en inferencia local: $e";
    }
  }

  // 3. Liberación de memoria
  void dispose() {
    _llmInference?.close();
    _llmInference = null;
  }
}

===========================================================================
⚠️ ANÁLISIS AUTOCRÍTICO (NOTAS DE DISEÑO)
===========================================================================

* CALENTAMIENTO (Thermal Throttling): La inferencia continua en GPU calienta 
  el dispositivo. Para un sistema de inventario, las ráfagas son cortas 
  (consultas), pero debemos evitar loops infinitos de generación.

* LATENCIA INICIAL: La primera vez que se llama a 'initializeEngine', puede 
  tardar de 3 a 8 segundos en cargar los pesos a la GPU. Se recomienda 
  precargar el modelo en el 'SplashScreen' o después del Login.

* CUANTIZACIÓN: Este código asume que el archivo .bin es la versión INT4. 
  Si intentas cargar la versión FP16 (original sin comprimir), la app 
  lanzará un 'Out of Memory' (OOM) en el 90% de los teléfonos.
