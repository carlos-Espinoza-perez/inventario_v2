===========================================================================
🎯 OBJETIVO DE ESTE MÓDULO
===========================================================================
Configurar el "System Prompt" de Gemma 2B para que actúe como un despachador 
lógico. Su misión es analizar el mensaje del usuario y clasificarlo en una 
categoría técnica (Intent) antes de procesar cualquier dato.

===========================================================================
🧠 LAS 3 INTENCIONES MAESTRAS (INTENTS)
===========================================================================

1. QUERY (Consulta): El usuario busca información. 
   Ej: "¿Hay stock de Coca Cola?", "¿Cuánto vendí ayer?".
   
2. ACTION (Acción): El usuario ordena una transacción.
   Ej: "Vende 2 jugos", "Registra entrada de 50 camisas".

3. CHAT (Conversación): Saludos o dudas generales.
   Ej: "Hola", "¿Qué puedes hacer?".

===========================================================================
🏗️ IMPLEMENTACIÓN DEL PROMPT DE SISTEMA (DART)
===========================================================================

class IntentClassifier {
  static const String systemInstruction = '''
Eres el cerebro del Sistema de Inventario de Carlos.
Tu salida DEBE ser exclusivamente un JSON con este formato:
{
  "intent": "QUERY" | "ACTION" | "CHAT",
  "entities": { "producto": "nombre", "cantidad": 0, "cliente": "nombre" },
  "confidence": 0.0-1.0
}

REGLAS:
- Si el usuario menciona cantidades y verbos imperativos (vende, agrega, quita), usa ACTION.
- Si pregunta por existencias o precios, usa QUERY.
- No saludes a menos que el intent sea CHAT.
''';

  Future<Map<String, dynamic>> analyze(String userInput) async {
    // 1. Unimos la instrucción con el mensaje del usuario
    final prompt = "$systemInstruction\nUsuario dice: $userInput";
    
    // 2. Llamamos al motor de Gemma (Paso 02)
    final rawResponse = await AIService().generateResponse(prompt);
    
    // 3. Parseamos el JSON generado por la IA
    try {
      return jsonDecode(rawResponse);
    } catch (e) {
      return {"intent": "CHAT", "content": rawResponse};
    }
  }
}

⚠️ ANÁLISIS AUTOCRÍTICO (REVISIÓN SENIOR)
===========================================================================

* ALUCINACIONES: Gemma 2B es pequeña y a veces puede intentar "inventar" 
  campos en el JSON. Por eso, el código de Dart DEBE validar que el JSON 
  tenga los campos obligatorios antes de proceder.

* FECHAS Y TIEMPOS: Las IAs offline no saben qué día es hoy. Es vital 
  inyectar la fecha actual en el prompt: "Hoy es Domingo 05 de Abril 2026".

* IDIOMA: Aunque Gemma entiende inglés mejor, forzaremos las respuestas 
  en español para mantener la consistencia con tu app de inventario.
